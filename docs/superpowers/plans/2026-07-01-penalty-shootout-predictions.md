# Penalty Shootout Predictions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let players predict who advances on penalties for a knockout match that ends level, and make that "advancer" the outcome that decides their 4/3/2/0 points.

**Architecture:** A nullable `penalty_winner` enum (`home`/`away`) is added to both `predictions` and `fixtures`. `ScoringService` gains an `outcome` helper — the higher score, or the shootout winner on a level score — and gates points on the predicted vs actual advancer matching. Validations keep the column consistent (only on knockout draws / level knockout results). A small Stimulus controller reveals a winner picker on both the player and admin cards the moment the two scores are equal.

**Tech Stack:** Rails 8.1, Minitest + Rails fixtures, Hotwire (Turbo + Stimulus via importmap), Tailwind/daisyUI, Solid Queue.

## Global Constraints

- Ruby version pinned by `.ruby-version`; Rails 8.1.
- All scoring math lives in `ScoringService`; models, controllers and views must never compute points themselves.
- `penalty_winner` is stored as a **side** (`home`/`away`), never a team id.
- The feature applies to **knockout** fixtures only (`stage` ≠ `group`); group behaviour must be byte-for-byte unchanged.
- Enums for `penalty_winner` MUST use `prefix: true, scopes: false` to avoid colliding with `home_*`/`away_*` methods and `ActiveRecord::Relation` scopes.
- Tests are Minitest with `fixtures :all`; run a single file with `bin/rails test <path>` and a single test with `bin/rails test <path> -n "/<substring>/"`.
- Commit after every green step (TDD).

```ruby
    def points_for(predicted_home:, predicted_away:, actual_home:, actual_away:,
                   predicted_pen_winner: nil, actual_pen_winner: nil)
      # A knockout that ends level is decided by the shootout winner, so the
      # advancer — not the drawn scoreline — is the outcome that must match.
      # For group matches (pen winner always nil) a level score stays a :draw,
      # so this reproduces the original win/draw/loss scoring exactly.
      return 0 unless outcome(predicted_home, predicted_away, predicted_pen_winner) ==
                      outcome(actual_home, actual_away, actual_pen_winner)
      return EXACT_POINTS if predicted_home == actual_home && predicted_away == actual_away
      return DIFFERENCE_POINTS if predicted_home - predicted_away == actual_home - actual_away

      TENDENCY_POINTS
    end
```

Add this private helper inside the `class << self` block (e.g. just before `champion_team_id`):

```ruby
      # The team that goes through: the higher score, or — on a level score —
      # the shootout winner (nil pen winner means a genuine group-stage draw).
      def outcome(home, away, pen_winner)
        return :home if home > away
        return :away if away > home

        pen_winner.presence&.to_sym || :draw
      end
```

    end
  end

```javascript
import { Controller } from "@hotwired/stimulus"

// Reveals the "who advances on penalties" picker whenever the two score inputs
// are level. Attached only to knockout fixtures. Hidden radios are disabled so
// a stale winner is never submitted; visible radios are required so a draw
// always names an advancer.
export default class extends Controller {
  static targets = ["home", "away", "picker", "option"]

  connect() {
    this.refresh()
  }

  refresh() {
    const home = this.homeTarget.value
    const away = this.awayTarget.value
    const level = home !== "" && away !== "" && Number(home) === Number(away)

    this.pickerTarget.classList.toggle("hidden", !level)
    this.optionTargets.forEach((option) => {
      option.disabled = !level
      option.required = level
    })
  }
}
```
